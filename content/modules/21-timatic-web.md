## Learning Objectives
- Access and navigate Timatic Web through the Amadeus Selling Platform
- Compare Timatic Web advantages over cryptic Timatic queries
- Use Timatic Web for complex multi-stop itineraries

## Overview
Timatic Web is the graphical, web-based version of Timatic that runs inside the Amadeus Selling Platform. While Module 20 covered cryptic command access, Timatic Web provides a more visual, user-friendly interface with the same data. It's easier to navigate for complex itineraries and shows information in a structured, readable format — helpful when you need to explain requirements to a customer.

## Key Concepts

### Concept 1: Timatic Web Interface
Accessed through the Amadeus Selling Platform's integrated tools or via [timaticweb.com](https://www.timaticweb.com/). It presents a form where you enter: nationality, destination, transit countries, and document types. Results appear in a structured, categorized layout.

### Concept 2: Advantages Over Cryptic
Timatic Web shows results in organized sections (Passport, Visa, Health, Customs, Currency) with clear headings. It's easier to read than the cryptic text output. For complex itineraries with multiple transits, the web version handles multi-country checks in one query.

### Concept 3: Pre-Built Itinerary Check
You can enter the full itinerary (origin → transit 1 → transit 2 → destination) and Timatic Web checks every point in one go. It flags any issues — "Transit visa required for Country X" — so nothing is missed.

### Concept 4: Country Information Sections
Beyond visa/passport, Timatic Web shows: currency import/export limits, customs restrictions (alcohol, tobacco, electronics), airport transit rules, and embassy contact information. Useful for providing comprehensive travel advice.

### Concept 5: Keeping Updated
Timatic Web shows the "last updated" date for each country's entry. Immigration rules change frequently — especially post-COVID health rules and visa policy updates. Always note the update date when advising customers.

### Concept 6: Sharing with Customers
Unlike cryptic output, Timatic Web results can be printed or saved as a summary to email to the customer. This gives them a written record of what documents they need — reducing "I didn't know" situations at the airport.

## Step-by-Step Guide
1. Open Timatic Web in the Amadeus Selling Platform (integrated tool) or via browser.
2. Enter passenger nationality (e.g., India).
3. Enter destination country (e.g., United Kingdom).
4. Add any transit countries (e.g., UAE for a Dubai connection).
5. Review all sections: Passport, Visa, Health, Transit.
6. Note any flags or warnings.
7. Share the summary with the customer and add relevant info to the PNR.

## Real-World Scenario
**Situation:** A family of four (Indian passports) is traveling Delhi → Dubai → London → Paris → Delhi. They need to know all document requirements for this complex itinerary.

**Question:** How do you check everything efficiently?

**Answer:** Open Timatic Web and enter: nationality = India, itinerary = DEL → DXB (transit) → LHR (destination) → CDG (destination) → DEL. The system checks: UAE transit (usually visa-free for short transits), UK entry (visa required for Indian nationals), France/Schengen entry (Schengen visa required), and return to India. Timatic Web flags that they need: a UK visa AND a Schengen visa (or a combined itinerary plan). It also checks passport validity for each country. Share the printout with the family so they can apply for visas well in advance.

## Common Mistakes
- **Mistake 1:** Checking only the final destination, not transits
  - *Why it matters:* Even Timatic Web requires you to enter transit countries — if you skip them, you miss transit visa requirements
  - *Correct approach:* Enter the COMPLETE itinerary including every transit point

- **Mistake 2:** Not checking the "last updated" date
  - *Why it matters:* If the entry was updated 6 months ago and the country recently changed visa policy, you might have stale info
  - *Correct approach:* Note the update date; for recent policy changes, verify with the embassy

- **Mistake 3:** Not providing the customer with a written summary
  - *Why it matters:* Verbal advice is forgotten — customer arrives at airport without the right documents
  - *Correct approach:* Print or email the Timatic Web summary so the customer has a reference

## Key Glossary Terms
- **Timatic Web:** The graphical, web-based version of the Timatic travel document database
- **Schengen Visa:** A visa valid for travel across 27 European countries in the Schengen Area
- **Transit Check:** Verifying document requirements for countries the passenger passes through, even without leaving the airport
- **IATA Travel Centre:** The public-facing version of travel document information maintained by IATA
- **Document Advisory:** A notification about changed requirements for a specific country
